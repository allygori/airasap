# Coding Guidelines

## TypeScript

- Use `type` over `interface`
- Avoid `any`

---

## Naming

- camelCase (variables)
- kebab-case (file name)
- PascalCase (components)

---

## Components

- Keep small and reusable
- Separate UI & logic

---

## Styling

- Tailwind CSS
- Shadcn only for dashboard

---

## Backend

- Thin controllers
- Logic in modules

---

## Error Handling

- Always return structured errors
