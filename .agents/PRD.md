# Product Requirements Document (PRD)

## 1. Overview
This project is a hybrid **Marketplace + Landing Page Platform** designed for the Indonesian market.

Core idea:
- Root = discovery marketplace (like Shopee/Tokopedia)
- Product detail = high-converting landing page (no distractions)
- Future-ready for AI agents (API-first communication)

---

## 2. Goals

### Phase 1 (MVP)
- Enable sellers to create product listings
- Render product detail pages as landing pages
- Basic checkout flow
- Simple analytics (views, clicks)

### Phase 2
- Landing page builder (section-based like Shopify)
- A/B testing system
- Conversion analytics
- AI-assisted product optimization

### Phase 3 (Future)
- AI agent-ready APIs
- Personal shopper agents
- Dynamic pricing & personalization

---

## 3. Target Users

### Sellers
- MSMEs (Indonesia)
- Dropshippers
- Brand owners

### Buyers
- Mobile-first users
- Price-sensitive
- Influenced by visual + trust signals

---

## 4. Key Features

### Marketplace (Discovery Layer)
- Product listing
- Search & filtering
- Categories

### Landing Page Mode
- No distractions
- High conversion UI
- Custom sections (JSON-driven layout)

### Seller Dashboard
- Product management
- Analytics
- A/B testing (future)

---

## 5. Core Differentiation

- Marketplace + Landing Page hybrid
- AI-agent friendly data structure
- Section-based layout system
- Conversion-first UX

---

## 6. Metrics

- Conversion Rate (CR)
- Click-through Rate (CTR)
- Average Order Value (AOV)
- Seller Retention

---

## 7. Constraints

- Monolithic architecture (Next.js)
- MongoDB (flexible schema for layout system)
- Fast page load (critical)

---

## 8. Risks

- Competing with large marketplaces
- Seller acquisition
- Performance bottlenecks from dynamic pages

---

## 9. Future Expansion

- Microservices architecture
- AI agents integration
- Multi-channel commerce
