
# TypeScript Conventions or Code Style

## Naming
-  Use kebab-case for file
- PascalCase for classes/types/interfaces
- camelCase for variables/functions

## Patterns
- Prefer functional patterns and immutability where possible


- **Naming**: Use kebab-case for file, PascalCase for classes/types/interfaces, camelCase for variables/functions.
- **Patterns**: Prefer functional patterns and immutability where possible, use Type instead of Interface.
- Always use const instead of let.
- Never use var.
- Use type instead of interface when possible.
- Use strict null checks.
